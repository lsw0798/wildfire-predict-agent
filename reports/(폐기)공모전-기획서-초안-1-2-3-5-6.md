## 1. 제품 및 서비스 개요

> 300자 이내 요약

본 서비스는 산림공공·빅데이터와 실시간 산불 정보를 결합해 특정 위치의 산불 위험도, 판단 근거, 오탐 가능성, 대응 행동을 웹에서 즉시 제시하는 AI 의사결정 지원 서비스이다. 지자체·현장 담당자의 초기 판단 시간을 줄이고 데이터 기반 예방·대응 체계를 강화하는 것을 목표로 한다.

## 2. 산림공공·빅데이터 활용 적정성

### 2.2 산림공공·빅데이터 활용 적정성

본 서비스의 핵심 기능은 “특정 좌표가 왜 위험한지, 현재 판단을 얼마나 신뢰할 수 있는지, 오탐 가능성은 무엇인지, 다음 행동은 무엇인지”를 설명하는 것이다. 이를 위해서는 단순한 산불 발생 건수보다 **위치·시간·원인·피해·기상·지형·접근성**이 결합된 데이터가 필요하다. `WSQ000301`은 발생 좌표, 발생 시점, 원인, 피해면적, 진화시간, 기상 요소, 임상·지형·거리 변수를 함께 포함하므로 좌표/반경 기반 위험요인 해석에 적합하다. 서비스는 입력 좌표 주변의 과거 산불을 조회해 인접 이력 수, 최신 발생연도, 주요 원인, 과거 평균 습도·풍속·기온 등을 산출하고 이를 위험도와 설명 근거로 전환한다.

`FRT000101` 산불발생위치도_전국은 공간데이터 형식으로 제공되어 지도 기반 서비스 확장성이 높다. 현재 분석의 중심은 CSV 기반 반경 조회이지만, 위치도 데이터는 행정구역별 산불 밀도, 위험지역 히트맵, 공간 클러스터링, 과거 발생 위치 검증에 활용할 수 있다. 따라서 공모전 서비스가 단순 텍스트 분석에 머무르지 않고 웹 지도·관제 화면으로 확장되는 근거 데이터로 적정하다.

실시간성 측면에서는 `todayFireGet` API를 활용한다. 과거 데이터만 사용할 경우 현재 산불 상태, 당일 기상성 위험 신호, 현장 변화가 반영되지 않는 한계가 있다. 본 서비스는 API가 정상 응답하면 실시간 습도·풍속·상태를 우선 적용하고, API key 누락 또는 호출 실패 시에는 fallback 결과와 데이터 품질 안내를 제공한다. 이는 재난 서비스에 필요한 **가용성**, **최신성**, **설명 가능성**을 동시에 높인다.

또한 AI Hub JSON 샘플과 자체 전처리 데이터는 모델·UI·데모 구현의 연결고리 역할을 한다. AI Hub 샘플은 기상, 지형, 연료, 인구·취약계층, 소방서 거리, 중요 인프라 거리, 도로 접근성 등 의사결정 feature를 구조화하여 위험 판단 로직 설계에 적합하다. 자체 전처리 산출물은 원천 데이터의 컬럼·행수·샘플을 빠르게 확인하게 하고, API 장애나 데이터 결측 시에도 분석 흐름이 끊기지 않도록 한다. 따라서 본 데이터 구성은 산림공공·빅데이터의 활용 목적과 서비스 기능이 직접 연결되어 활용 적정성이 높다.

## 3. 독창성(차별성)

### 3.1 기존 유사 서비스와의 차별점

기존 산불 관련 서비스는 주로 **현재 산불 발생 현황**, **위험등급 지도**, **국민 행동요령**, **신고/알림** 기능을 제공한다. 예를 들어 산림청 국가산불정보시스템은 실시간 산불 발생 현황, 산불 통계, 산불위험등급 등을 제공하고, 국립산림과학원의 국가산불위험예보시스템은 산불위험 예보 정보를 제공한다. 국민안전24 등 재난안전 서비스는 재난 행동요령과 안전 정보를 제공한다.

본 서비스는 이러한 정보 제공형 서비스에서 한 단계 나아가, 사용자가 지정한 좌표와 반경을 기준으로 **“왜 위험한가, 얼마나 믿을 수 있는가, 오탐 가능성은 무엇인가, 지금 무엇을 해야 하는가”**까지 하나의 분석 흐름으로 제시한다는 점이 차별적이다.

| 비교 항목 | 기존 서비스 경향 | 본 서비스 차별점 |
|---|---|---|
| 정보 단위 | 전국/시도 단위 현황, 위험등급, 통계 중심 | 사용자가 지정한 좌표·반경 단위의 현장형 분석 |
| 분석 방식 | 발생 현황 및 예보 정보 제공 중심 | 과거 이력 + 실시간 상태 + AI/agent source selection 기반 판단 흐름 |
| 설명 방식 | 등급 또는 발생 정보 중심 | XAI 요약, 주요 원인, confidence, confidence margin, 데이터 품질 안내 제공 |
| 오탐 대응 | 별도 오탐 검토 UI가 제한적 | false-positive review를 통해 오탐 가능성과 검토 신호를 함께 제시 |
| 사용자 행동 | 신고/행동요령 안내 중심 | 공무원, 일반 사용자 등 사용자 유형에 따른 권장 행동 체크리스트 제공 |
| 운영 흐름 | 사용자가 정보를 찾아 해석 | Top 10 관제 우선순위 클릭 시 `/analyze` 자동 실행으로 판단 흐름 단축 |

### 3.2 서비스만의 창의적·독창적 요소

1. **관제 우선순위 Top 10 → 자동 분석 연결**  
   과거 산불 발생 빈도와 위험 신호를 기반으로 우선 확인해야 할 지점을 카드 형태로 제시하고, 사용자가 Top 10 카드를 클릭하면 분석 페이지에서 자동으로 위험 분석이 실행된다. 이는 단순 대시보드가 아니라 **관제 상황판에서 분석 행동까지 이어지는 UX**이다.

2. **Top 10과 Top 5의 역할 분리**  
   Top 10은 즉시 분석이 필요한 관제 우선순위, Top 5는 좌표 기반 현장 확인용으로 구분했다. 이를 통해 자동 분석과 사용자 최종 확인 단계를 분리하여 운영 안전성을 높였다.

3. **LLM 의존도를 낮춘 hybrid 구조**  
   OpenAI API가 사용 가능하면 LLM이 historical/realtime source 선택을 보조하지만, API key 또는 credit 문제가 있어도 rule fallback으로 historical + realtime 경로를 계속 사용한다. 따라서 AI 기능을 사용하되, 공공 서비스에 필요한 안정성을 확보했다.

4. **위험도와 함께 불확실성을 표시**  
   단순히 “높음/보통/낮음”만 표시하지 않고 confidence, confidence margin, 데이터 품질 상태, 추정 field를 함께 제공한다. 재난 의사결정에서 중요한 “판단의 근거와 한계”를 사용자가 확인할 수 있다.

5. **오탐 가능성 검토를 UI에 포함**  
   산불 탐지·예측 서비스는 과잉 경보가 반복되면 신뢰도가 떨어진다. 본 서비스는 false-positive panel을 통해 오탐 가능성, 검토한 신호, 추가 확인 필요 사항을 함께 제공하여 실제 현장 수용성을 높인다.

### 5.4 홍보·마케팅 및 사업 활성화 방안

1. **공공 PoC 중심 확산**  
   산불 다발 지역 지자체를 대상으로 봄철·가을철 산불조심기간 전후 PoC를 진행하고, 관제 우선순위와 대응 리포트를 제공한다.

2. **산림·재난안전 박람회 및 공모전 연계**  
   산림청, 지자체, 재난안전 관련 행사에서 데모 화면과 실제 데이터 기반 분석 리포트를 시연한다.

3. **지역 맞춤 위험 리포트 제공**  
   특정 시군구별 “최근 10년 산불 원인·위험지점·대응 우선순위” 리포트를 무료 샘플로 제공해 유료 구독 전환을 유도한다.

4. **민간 파트너십**  
   보험사, 관광·레저 사업장, 산림조합, 드론 관제 업체와 협력하여 API 또는 대시보드 형태로 서비스 범위를 확장한다.

5. **공공데이터 활용성과 강조**  
   산림빅데이터 플랫폼 데이터와 실시간 API를 활용한 사례로 홍보하여 공공데이터 기반 디지털 전환 사례로 포지셔닝한다.

## 6. 사회적 가치 창출(기대효과)

### 6.1 국민편의 및 사회문제 해결 효과

본 서비스는 산불 발생 후 대응뿐 아니라, **사전 위험 감지와 초기 의사결정 지원**에 초점을 둔다. 사용자는 특정 위치의 위험도, 판단 근거, 불확실성, 오탐 가능성, 대응 행동을 한 화면에서 확인할 수 있다. 이를 통해 재난 담당자는 여러 시스템을 오가며 정보를 해석하는 시간을 줄이고, 주민·관광객·산림 작업자에게 더 빠르고 일관된 대응 안내를 제공할 수 있다.

정량적 기대효과는 다음과 같이 설정할 수 있다.

| 기대효과 | 정량 지표 예시 | 산정 방식 |
|---|---|---|
| 초기 판단 시간 단축 | 위험 위치 1건당 정보 확인 시간 10~20분 절감 목표 | 기존 수동 조회 대비 Top 10 자동 분석 및 요약 카드 활용 |
| 고위험지 우선 점검 | 관할 구역별 Top 10 감시 지점 상시 제공 | 과거 이력·최근성·원인 기반 우선순위 카드 운영 |
| 오탐 대응 부담 완화 | 불확실성·오탐 검토 항목 제공률 100% 목표 | 모든 분석 결과에 false-positive review와 confidence 표시 |
| 데이터 기반 보고 자동화 | 지역별 산불 위험 리포트 작성 시간 절감 | 분석 결과와 근거 데이터를 보고서/대시보드로 재사용 |
| 현장 대응 품질 향상 | 사용자 유형별 행동 가이드 제공 | 공무원/일반 사용자 등 대상별 권장 행동 체크리스트 제공 |

### 6.2 정성적 기대효과

1. **재난 대응의 투명성 향상**  
   위험도만 제시하는 것이 아니라 “왜 그런 판단이 나왔는지”를 XAI 요약과 supporting evidence로 제공해 담당자와 시민 모두가 판단 근거를 이해할 수 있다.

2. **공공데이터 활용 가치 증대**  
   산림빅데이터 플랫폼의 산불 발생 이력, 위치도, API를 실제 서비스 흐름에 연결하여 공공데이터가 행정·현장 의사결정에 직접 활용되는 사례를 만든다.

3. **지역 안전 격차 완화**  
   대형 관제 시스템 구축이 어려운 중소 지자체도 웹 기반 SaaS 형태로 위험 분석 기능을 활용할 수 있어 지역 간 산불 대응 역량 격차를 줄일 수 있다.

4. **과잉 경보로 인한 신뢰 저하 방지**  
   false-positive review와 confidence margin을 함께 표시해 무조건적인 경보가 아니라 검토 가능한 위험 판단을 제공한다. 이는 현장 담당자의 서비스 수용성을 높인다.

5. **기후위기 대응형 산림 관리 기반 마련**  
   건조·강풍·고온 등 산불 위험 요인이 증가하는 상황에서, 축적된 산림공공데이터와 실시간 데이터를 결합해 예방 중심의 산림 관리 체계를 강화할 수 있다.
