산불 에이전트 아키텍처

산불 의사결정 지원 웹서비스의 전체 구조 · 사용자 입력 → 웹 화면 → 분석 API/워크플로 → 과거 이력·실시간 데이터·소스 선택 → 설명 가능한 결과 반환

입력 계층

  • • 사용자는 좌표, 분석 반경, 사용자 유형을 입력한다.
  • • 웹 화면은 카카오 지도와 결과 카드 UI를 담당한다.
  • • `/api/analyze`에 JSON 요청 본문을 전송한다.

분석 계층

  • • 워크플로 엔진이 조회 → 특징 생성 → 위험도 평가 → 검증/보정을 순차 실행한다.
  • • 실제 위험도, 오탐, 신뢰도 계산은 규칙 기반 서비스가 수행한다.
  • • 결과는 설명 가능한 보고서 형태로 조립된다.

데이터·LLM 계층

  • • 과거 CSV는 반경 내 산불 이력과 평균 특징 추정에 사용된다.
  • • 실시간 API는 습도, 풍속, 온도 등 상태값을 제공한다.
  • • LLM은 전체 분석이 아니라 조회 소스 선택 단계에서만 사용된다.
