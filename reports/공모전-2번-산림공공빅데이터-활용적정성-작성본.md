### 2.2 산림공공·빅데이터 활용 적정성

본 서비스의 핵심 기능은 “특정 좌표가 왜 위험한지, 현재 판단을 얼마나 신뢰할 수 있는지, 오탐 가능성은 무엇인지, 다음 행동은 무엇인지”를 설명하는 것이다. 이를 위해서는 단순한 산불 발생 건수보다 **위치·시간·원인·피해·기상·지형·접근성**이 결합된 데이터가 필요하다. `WSQ000301`은 발생 좌표, 발생 시점, 원인, 피해면적, 진화시간, 기상 요소, 임상·지형·거리 변수를 함께 포함하므로 좌표/반경 기반 위험요인 해석에 적합하다. 서비스는 입력 좌표 주변의 과거 산불을 조회해 인접 이력 수, 최신 발생연도, 주요 원인, 과거 평균 습도·풍속·기온 등을 산출하고 이를 위험도와 설명 근거로 전환한다.

`FRT000101` 산불발생위치도_전국은 공간데이터 형식으로 제공되어 지도 기반 서비스 확장성이 높다. 현재 분석의 중심은 CSV 기반 반경 조회이지만, 위치도 데이터는 행정구역별 산불 밀도, 위험지역 히트맵, 공간 클러스터링, 과거 발생 위치 검증에 활용할 수 있다. 따라서 공모전 서비스가 단순 텍스트 분석에 머무르지 않고 웹 지도·관제 화면으로 확장되는 근거 데이터로 적정하다.

실시간성 측면에서는 `todayFireGet` API를 활용한다. 과거 데이터만 사용할 경우 현재 산불 상태, 당일 기상성 위험 신호, 현장 변화가 반영되지 않는 한계가 있다. 본 서비스는 API가 정상 응답하면 실시간 습도·풍속·상태를 우선 적용하고, API key 누락 또는 호출 실패 시에는 fallback 결과와 데이터 품질 안내를 제공한다. 이는 재난 서비스에 필요한 **가용성**, **최신성**, **설명 가능성**을 동시에 높인다.

또한 AI Hub JSON 샘플과 자체 전처리 데이터는 모델·UI·데모 구현의 연결고리 역할을 한다. AI Hub 샘플은 기상, 지형, 연료, 인구·취약계층, 소방서 거리, 중요 인프라 거리, 도로 접근성 등 의사결정 feature를 구조화하여 위험 판단 로직 설계에 적합하다. 자체 전처리 산출물은 원천 데이터의 컬럼·행수·샘플을 빠르게 확인하게 하고, API 장애나 데이터 결측 시에도 분석 흐름이 끊기지 않도록 한다. 따라서 본 데이터 구성은 산림공공·빅데이터의 활용 목적과 서비스 기능이 직접 연결되어 활용 적정성이 높다.
