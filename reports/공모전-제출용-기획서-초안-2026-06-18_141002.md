# 공모전 제출용 기획서 초안

## 프로젝트 제목

# **FireAgent: AI 에이전트 기반 산불 예측·감지·의사결정 자동화 서비스**

## 한 줄 소개

사용자가 위치와 반경을 입력하면 AI 에이전트가 산림공공·빅데이터와 실시간 정보를 결합해 산불 위험도, 오탐 가능성, 판단 근거, 대응 행동을 한 번에 제시하는 현장형 의사결정 지원 웹서비스

---

## 1. 제품 및 서비스 개요

> 300자 이내 요약

본 서비스는 산불 예측·감지·의사결정에 AI 에이전트를 적용한 공공 재난 대응 웹서비스이다. 사용자가 위치와 반경을 입력하면 과거 산불 이력, 실시간 공공데이터, 위치 맥락, 사용자 유형을 종합해 위험도, 오탐 가능성, XAI 기반 판단 근거와 대응 가이드를 자동 제공한다.

---

## 2. 산림공공·빅데이터 활용 적정성

본 서비스는 산불 예측·탐지·의사결정 지원에 필요한 **과거 발생 이력**, **공간 위치 정보**, **실시간 산불 상태**, **기상·지형·연료·접근성 feature**, **데이터 품질 및 불확실성 판단 근거**를 산림공공·빅데이터로 구성한다. 산림빅데이터 플랫폼의 산불 발생 경향성 데이터와 전국 산불 발생 위치도는 좌표·반경 기반 과거 이력 분석의 핵심 근거로 활용하고, 산림청 오늘 산불 API는 당일·실시간 상태를 보완한다. 또한 AI Hub 산불 대응 관련 JSON 샘플과 자체 전처리 산출물을 결합하여 위험도, confidence, XAI 설명, 오탐 검토, fallback 판단까지 하나의 서비스 흐름으로 연결한다. **[수정 반영] 최근에는 여기에 더해 모델 점수와 규칙 기반 점수의 불일치, 모델 feature coverage 부족, 낮은 모델 confidence를 별도 신뢰도 진단 근거로 반영하여, 단순 조회형이 아니라 ‘저신뢰 원인 분석 → 보수적 재검토 → 결과 보정’ 흐름까지 포함하도록 고도화했다.**

### 2.2 산림공공·빅데이터 활용 적정성

본 서비스의 핵심 기능은 “특정 좌표가 왜 위험한지, 현재 판단을 얼마나 신뢰할 수 있는지, 오탐 가능성은 무엇인지, 다음 행동은 무엇인지”를 설명하는 것이다. 이를 위해서는 단순한 산불 발생 건수보다 **위치·시간·원인·피해·기상·지형·접근성**이 결합된 데이터가 필요하다. `WSQ000301`은 발생 좌표, 발생 시점, 원인, 피해면적, 진화시간, 기상 요소, 임상·지형·거리 변수를 함께 포함하므로 좌표/반경 기반 위험요인 해석에 적합하다.

서비스는 입력 좌표 주변의 과거 산불을 조회해 인접 이력 수, 최신 발생연도, 주요 원인, 과거 평균 습도·풍속·기온 등을 산출하고 이를 위험도와 설명 근거로 전환한다. `FRT000101` 산불발생위치도_전국은 공간데이터 형식으로 제공되어 지도 기반 서비스 확장성이 높다. 현재 분석의 중심은 CSV 기반 반경 조회이지만, 위치도 데이터는 행정구역별 산불 밀도, 위험지역 히트맵, 공간 클러스터링, 과거 발생 위치 검증에 활용 가능하다.

실시간성 측면에서는 `todayFireGet` API를 활용한다. 과거 데이터만 사용할 경우 현재 산불 상태, 당일 기상성 위험 신호, 현장 변화가 반영되지 않는 한계가 있다. 본 서비스는 API가 정상 응답하면 실시간 습도·풍속·상태를 우선 적용하고, API key 누락 또는 호출 실패 시에는 fallback 결과와 데이터 품질 안내를 제공한다. 이는 재난 서비스에 필요한 **가용성**, **최신성**, **설명 가능성**을 동시에 높인다.

향후 개인정보가 포함될 수 있는 통신·카드·신고 데이터는 개인 식별정보를 제거하고 격자·행정동·시간대 단위로 집계하여 활용한다. 이를 통해 공공성과 안전성을 유지하면서도 산림공공데이터 중심의 서비스가 재난 대응, 관광·레저 안전관리, 보험·시설 위험관리 등 다양한 분야로 확장될 수 있다.

## 3. 독창성(차별성)

### 3.1 기존 유사 서비스와의 차별점

기존 산불 관련 서비스는 주로 **현재 산불 발생 현황**, **위험등급 지도**, **국민 행동요령**, **신고/알림** 기능을 제공한다. 본 서비스는 이러한 정보 제공형 서비스에서 한 단계 나아가, 사용자가 지정한 좌표와 반경을 기준으로 **“왜 위험한가, 얼마나 믿을 수 있는가, 오탐 가능성은 무엇인가, 지금 무엇을 해야 하는가”**까지 하나의 분석 흐름으로 제시한다는 점이 차별적이다.

| 비교 항목 | 기존 서비스 경향 | 본 서비스 차별점 |
|---|---|---|
| 정보 단위 | 전국/시도 단위 현황, 위험등급, 통계 중심 | 사용자가 지정한 좌표·반경 단위의 현장형 분석 |
| 분석 방식 | 발생 현황 및 예보 정보 제공 중심 | 과거 이력 + 실시간 상태 + AI 에이전트 source selection 기반 판단 흐름 |
| 판단 구조 | 단일 모델 또는 고정 규칙이 산출한 결과를 사용자가 해석 | 에이전트 루프가 조회, 검토, 보정, 설명 생성을 단계적으로 수행 |
| 오탐 대응 | 안개·구름·연무 등으로 인한 오탐 검토 기능이 제한적 | false-positive review를 통해 오탐 가능성과 검토 신호를 함께 제시 |
| 설명 방식 | 등급 또는 발생 정보 중심 | XAI 요약, 주요 원인, confidence, confidence margin, 데이터 품질 안내 제공 |
| 사용자 행동 | 신고/행동요령 안내 중심 | 사용자 유형에 따른 권장 행동 체크리스트와 초기 의사결정 가이드 제공 |
| 운영 안정성 | 특정 데이터나 모델 장애 시 서비스 품질 저하 가능 | LLM/API 장애 시 rule fallback과 데이터 품질 안내로 분석 흐름 유지 |

### 3.2 서비스만의 창의적·독창적 요소

1. **산불 대응에 AI 에이전트 구조 도입**  
   단순 예측 모델이 아니라, 데이터 조회 → 품질 검토 → 오탐 가능성 판단 → 위험도 보정 → 대응 가이드 생성까지 수행하는 에이전트형 의사결정 체계를 적용한다.

2. **오탐 가능성의 별도 판단 지표화**  
   기존 탐지 모델의 주요 한계인 안개·구름·연무 오인 문제를 해결하기 위해, 산불 위험도와 별개로 false-positive risk를 함께 제시한다.

3. **공공데이터와 현장 맥락의 결합**  
   실시간 공공데이터뿐 아니라 과거 산불 이력, 위치 반경, 주변 환경, 사용자 유형을 함께 반영하여 동일한 위험도라도 상황에 맞는 다른 판단과 대응 안내를 제공한다.

4. **설명 가능한 재난 의사결정 지원**  
   담당자가 단순 점수를 해석하는 방식에서 벗어나, AI가 판단 근거와 불확실성을 설명함으로써 공공 의사결정의 신뢰성과 책임성을 높인다.

5. **[수정 반영] 저신뢰 원인 분석과 재라우팅 구조**  
   단순히 결과만 제시하지 않고, 모델과 규칙 기반 판단이 충돌하거나 모델 coverage가 낮을 때 그 이유를 별도 진단하고 conservative reroute를 적용한다. 이는 “신뢰도가 낮으면 왜 낮은지 설명하고, 다른 경로로 다시 검토한다”는 에이전트형 흐름을 강화한다.

6. **사용자 유형별 대응 자동화**  
   지자체 담당자, 산림 관리 주체, 재난 대응 인력, 시민 등 사용자 역할에 따라 필요한 행동 가이드를 다르게 제시하여 현장 적용성을 강화한다.

7. **예측·감지·판단·행동의 통합 서비스화**  
   기존 서비스가 위험 정보 제공에 머무르는 것과 달리, 본 서비스는 “현재 위험한가?”, “탐지가 오탐일 가능성은 있는가?”, “무엇을 먼저 조치해야 하는가?”까지 연결하는 의사결정 자동화 플랫폼을 지향한다.

## 4. 기술성

### 4.1 서비스 구성 및 기능

본 서비스는 웹 기반으로 구현되는 산불 의사결정 지원 시스템이며, 사용자 입력부터 분석 결과 제공까지 다음과 같은 구조로 동작한다.

| 구성 요소 | 적용 기술/구조 | 주요 기능 |
|---|---|---|
| Frontend | Next.js, TypeScript, Kakao Map UI | 위치·반경 입력, 지도 기반 좌표 선택, 분석 결과 카드, 오탐 검토 패널, 대응 가이드 표시 |
| Backend API | FastAPI, Python | `/api/analyze`, `/health`, 분석 요청 검증, 분석 orchestration 호출 |
| Agent Workflow | LangGraph | 조회 소스 선택, 과거·실시간 데이터 조회, 위험도 산정, 오탐 검토, 데이터 품질 검토, 최종 응답 생성 |
| Historical Service | 산불 발생 경향성 CSV, 반경 기반 거리 계산 | 입력 좌표 주변 과거 산불 이력 탐색, 주요 원인·최근 발생연도·인접 이력 수 산출 |
| Realtime Service | `todayFireGet` API | 실시간 산불 상태·습도·풍속 등 위험 신호 반영, 실패 시 fallback 처리 |
| Risk & XAI Layer | 규칙 기반 risk scoring, confidence metrics, false-positive review | 위험도, confidence, confidence margin, 오탐 가능성, 판단 근거, 데이터 품질 요약 생성 |
| **[수정 반영] Low-confidence Routing Layer** | confidence diagnostics, reroute decision, conservative review | 저신뢰 원인 분해, reroute 여부 판단, model-rule disagreement/coverage 부족 시 보수 경로 전환 |
| Fallback Layer | rule fallback, source quality handling | OpenAI API 크레딧 소진, API 장애, 데이터 결측 상황에서도 최소 분석 결과 제공 |

### 4.2 핵심 기술 흐름

현재 분석 흐름은 LangGraph 기반 단계형 agent workflow로 구성된다.

1. **select_sources**  
   입력 위치, 사용자 유형을 바탕으로 historical/realtime 조회 필요성을 판단한다. OpenAI 사용 가능 시 LLM 기반 선택을 활용하고, API credit 또는 장애 상황에서는 rule fallback으로 계속 분석한다.

2. **fetch_historical**  
   입력 좌표와 반경(`radius_km`)을 기준으로 과거 산불 발생 이력을 검색한다. 기본 반경은 50km이며, 사용자가 지정한 반경이 있으면 해당 값을 사용한다.

3. **fetch_realtime**  
   `todayFireGet` API를 호출해 실시간 상태를 수집한다. API 실패 또는 key 문제 발생 시 fallback source로 표시하고, 이후 confidence와 데이터 품질 안내에 반영한다.

4. **derive_features**  
   과거 이력, 실시간 데이터, 추정값을 결합해 위험도 산정용 feature를 생성한다. 실시간 값이 있으면 우선 적용하고, 없으면 주변 과거 이력 평균값 또는 기본값을 사용한다.

5. **primary_assessment**  
   습도, 풍속, 가뭄일수, 경사, 연료 수분, 취약계층 지표 등을 바탕으로 위험 점수와 위험 등급을 산출한다. **[수정 반영] 이 단계에서는 규칙 기반 점수뿐 아니라 주입 가능한 baseline model scorer가 있을 경우 model score, model confidence, feature coverage도 함께 계산할 수 있다.**

6. **[수정 반영] diagnose_path / low-confidence routing**  
   1차 평가 결과와 source 품질을 함께 검토하여 reroute 여부를 판단한다. 과거/실시간 source 누락뿐 아니라, **model-rule score disagreement**, **낮은 model confidence**, **낮은 feature coverage**가 있으면 보수적 재검토 경로(`augment_both`)로 전환한다.

7. **augment_sources & secondary_analysis**  
   필요한 경우 historical/realtime source를 추가 보강 조회하고, 이후 secondary analysis에서 위험 점수와 false-positive risk를 재조정한다. **[수정 반영] 특히 모델 신호와 규칙 기반 판단이 크게 충돌하면 conservative penalty를 적용해 결과를 더 신중하게 해석한다.**

8. **verification_gate**  
   추가 검토가 필요한 조건을 판단한다. 예를 들어 추정값이 많거나, realtime source가 fallback이거나, 오탐 위험이 높으면 검토 단계로 진입한다.

9. **false_positive_review**  
   안개, 구름, 연무 등으로 인한 오탐 가능성을 가시성, 습도, 풍속, 표면온도, 화재 강도 신호와 함께 검토한다.

10. **data_quality_review / source_review**  
   데이터 출처가 live인지 fallback인지, 추정 field가 얼마나 포함되어 있는지 평가한다.

11. **decision_adjustment**  
   오탐 가능성, 데이터 품질, 소스 상태에 따라 위험 점수와 신뢰도 해석을 보정한다.

12. **compose_report / build_response**  
    최종적으로 위험도, 오탐 가능성, confidence, confidence margin, 분석 반경, 반경 경계 좌표, XAI 근거, 검토 신호, 데이터 품질 요약, 대응 행동을 응답한다. **[수정 반영] 이때 confidence_breakdown, confidence_reasons, reroute_applied, reroute_reason, model_decision_mode 같은 진단 정보도 함께 반환할 수 있다.**

### 4.3 핵심 기술적 특징

| 기술 특징 | 세부 내용 | 서비스 완성도 기여 |
|---|---|---|
| Agentic Source Selection | LLM 또는 rule fallback이 historical/realtime source 선택 | 불필요한 조회를 줄이고, 상황별 데이터 조합 가능 |
| LangGraph Workflow | 단일 함수가 아니라 단계형 그래프로 분석 구성 | 분석 흐름이 명확하고 확장 가능 |
| 반경 기반 분석 | `radius_km` 입력, 기본 50km, 북/남/동/서 radius point 계산 | 위치 기반 현장 분석과 지도 오버레이 가능 |
| False-positive Review | 가시성, 습도, 풍속, 표면온도, 화재 강도 신호 검토 | 안개·구름·연무 등 오탐 문제 대응 |
| Confidence Margin | confidence와 함께 ±오차 폭 및 source quality 표시 | 재난 판단의 불확실성을 투명하게 제공 |
| **[수정 반영] Low-confidence Reroute** | 모델-규칙 불일치, 낮은 model confidence, 낮은 feature coverage를 탐지해 보수 경로로 재검토 | 결과를 무조건 단정하지 않고 ‘낮은 신뢰도의 이유’를 설명 가능 |
| XAI 응답 | `risk_summary_text`, `xai_reasons`, `reviewed_signals`, `data_quality_summary` 제공 | 사용자가 결과를 직접 해석해야 하는 부담 완화 |
| Robust Fallback | OpenAI credit 소진, API 실패, 데이터 결측 시에도 fallback 응답 | 공공 서비스에 필요한 안정성 확보 |

### 4.4 시연 시나리오

#### 시나리오 1: 위치·반경 기반 산불 위험 분석

1. 사용자가 웹 서비스에 접속한다.
2. 지도에서 분석할 위치를 클릭하거나 위도·경도를 직접 입력한다.
3. 분석 반경을 1km, 3km, 5km, 10km, 20km 중 선택하거나 직접 입력한다.
4. 사용자 유형을 선택한다. 예: 공무원, 진압대원, 구조요원, 시민.
5. “분석 실행” 버튼을 클릭한다.
6. 백엔드는 `/api/analyze`로 요청을 받아 LangGraph agent workflow를 실행한다.
7. 화면에는 다음 결과가 표시된다.
   - 위험 수준 및 위험 점수
   - confidence 및 confidence margin
   - 반경 기반 분석 범위
   - 오탐 가능성
   - 주요 판단 근거(XAI)
   - 검토한 신호
   - 데이터 품질 요약
   - 대응 행동 가이드

#### 시나리오 2: 오탐 가능성 검토

1. 특정 위치에서 위험 신호가 감지된다.
2. 에이전트는 실시간 신호와 과거 이력을 함께 확인한다.
3. 습도, 가시성, 표면온도, 풍속, 화재 강도 등을 검토한다.
4. 안개·구름·연무로 인한 오탐 가능성이 있으면 false-positive risk를 함께 제공한다.
5. 사용자는 단순 “위험” 알림이 아니라, “추가 확인이 필요한 위험 판단”인지 확인할 수 있다.

#### 시나리오 3: OpenAI API 사용 불가 상황

1. OpenAI API credit 소진 또는 사용 한도 초과가 발생한다.
2. 시스템은 분석 전체를 중단하지 않고 `GROUP_E_ENABLED=false` 또는 rule fallback 모드로 전환한다.
3. historical + realtime source를 기본 조회하여 분석을 계속한다.
4. 응답에는 “LLM 소스 선택 기능이 비활성화되어 규칙 기반 판단으로 계속 분석”했다는 설명이 포함된다.
5. 공공·재난 서비스에서 중요한 장애 대응성과 운영 지속성을 확보한다.

## 5. 발전 가능성

### 5.1 시장성 및 실제 사업화 가능성

산불은 인명 피해, 산림자원 손실, 관광·지역경제 위축, 대기질 악화 등 광범위한 사회·경제적 손실을 유발한다. 특히 기후변화로 건조·강풍 조건이 빈번해지면서 산불 대응은 단순 사후 진화가 아니라 **사전 위험 예측, 취약지 우선 점검, 초기 의사결정 자동화** 중심으로 전환될 필요가 있다.

동시에 산불 예측·감지 서비스는 **오탐을 줄이고 판단 정확도를 높이는 것**이 사업화의 핵심이다. 위성·영상 기반 탐지는 연기와 구름의 시각적 유사성 때문에 구분이 어렵고, NASA NTRS의 “Smoke or Cloud” 연구도 연기와 구름 구분이 실시간 열점 탐지 정확도 개선에 중요하다고 설명한다. 또한 NASA FIRMS는 활성 열점 데이터에 산불뿐 아니라 농업 소각, 화산·지열 활동, 산업시설, 가스 플레어, 폐기물 소각장, 시멘트·철강·석유화학 시설 등 비산불 열원이 포함될 수 있어 추가 정보와 분석이 필요하다고 밝힌다. 민간 위성 산불 모니터링 기업 OroraTech 역시 태양광 패널, 철제 지붕, 수면 반사, 가열된 암석·수확지 등이 적외선 산불 신호를 모방해 false positive를 만들 수 있다고 설명한다. 따라서 실제 시장에서는 단순히 “위험 알림을 많이 보내는 서비스”보다, 과거 이력·실시간 기상·공간 맥락·confidence·false-positive review를 결합해 **불필요한 출동과 경보 피로를 줄이는 고정확도 의사결정 지원 서비스**의 수요가 크다.

본 서비스의 주요 고객은 다음과 같다.

| 고객군 | 니즈 | 제공 가치 |
|---|---|---|
| 지자체 산림·재난 부서 | 관할 구역 내 취약지 우선순위 파악, 주민 대피 판단 지원 | 관제 우선순위, 위치 기반 위험 분석, 대응 행동 체크리스트 |
| 산림청·공공기관 | 공공 데이터 기반 시범 분석, 정책·예방 계획 수립 | 데이터 기반 산불 위험 요인 분석, 지역별 위험 설명 자료 |
| 산림조합·산림경영체 | 산림 자산 피해 예방, 작업장 안전관리 | 산림자산 주변 위험도 모니터링, 작업 제한 판단 보조 |
| 관광지·리조트·캠핑장 | 방문객 안전, 운영 중단 판단, 화재 예방 | 주변 산림 위험도 대시보드, 알림·점검 시나리오 |
| 보험사·재보험사 | 산림·시설물 위험평가, 손해율 관리 | 위치 기반 위험 scoring API, 손해 예방 리포트 |

### 5.4 홍보·마케팅 및 사업 활성화 방안

1. **공공 PoC 중심 확산**  
   산불 다발 지역 지자체를 대상으로 봄철·가을철 산불조심기간 전후 PoC를 진행하고, 관제 우선순위와 대응 리포트를 제공한다.

2. **산림·재난안전 박람회 및 공모전 연계**  
   산림청, 지자체, 재난안전 관련 행사에서 데모 화면과 실제 데이터 기반 분석 리포트를 시연한다.

3. **지역 맞춤 위험 리포트 제공**  
   특정 시군구별 “최근 산불 원인·위험지점·대응 우선순위” 리포트를 무료 샘플로 제공해 유료 구독 전환을 유도한다.

4. **민간 파트너십**  
   보험사, 관광·레저 사업장, 산림조합, 드론 관제 업체와 협력하여 API 또는 대시보드 형태로 서비스 범위를 확장한다.

5. **공공데이터 활용성과 강조**  
   산림빅데이터 플랫폼 데이터와 실시간 API를 활용한 사례로 홍보하여 공공데이터 기반 디지털 전환 사례로 포지셔닝한다.

## 6. 사회적 가치 창출(기대효과)

### 6.1 국민편의 및 사회문제 해결 효과

본 서비스는 산불 발생 후 대응뿐 아니라, **사전 위험 감지와 초기 의사결정 지원**에 초점을 둔다. 사용자는 특정 위치의 위험도, 판단 근거, 불확실성, 오탐 가능성, 대응 행동을 한 화면에서 확인할 수 있다. 이를 통해 재난 담당자는 여러 시스템을 오가며 정보를 해석하는 시간을 줄이고, 주민·관광객·산림 작업자에게 더 빠르고 일관된 대응 안내를 제공할 수 있다.

정량적 기대효과는 다음과 같이 설정할 수 있다.

| 기대효과 | 정량 지표 예시 | 산정 방식 |
|---|---|---|
| 초기 판단 시간 단축 | 위험 위치 1건당 정보 확인 시간 10~20분 절감 목표 | 기존 수동 조회 대비 위치·반경 입력 후 자동 분석 및 요약 카드 활용 |
| 고위험지 우선 점검 | 관할 구역별 우선 감시 지점 상시 제공 | 과거 이력·최근성·원인 기반 우선순위 카드 운영 |
| 오탐 대응 부담 완화 | 불확실성·오탐 검토 항목 제공률 100% 목표 | 모든 분석 결과에 false-positive review와 confidence 표시 |
| 데이터 기반 보고 자동화 | 지역별 산불 위험 리포트 작성 시간 절감 | 분석 결과와 근거 데이터를 보고서/대시보드로 재사용 |
| 현장 대응 품질 향상 | 사용자 유형별 행동 가이드 제공 | 공무원/일반 사용자 등 대상별 권장 행동 체크리스트 제공 |

### 6.2 정성적 기대효과

1. **재난 대응의 투명성 향상**  
   위험도만 제시하는 것이 아니라 “왜 그런 판단이 나왔는지”를 XAI 요약과 supporting evidence로 제공해 담당자와 시민 모두가 판단 근거를 이해할 수 있다.

2. **공공데이터 활용 가치 증대**  
   산림빅데이터 플랫폼의 산불 발생 이력, 위치도, API를 실제 서비스 흐름에 연결하여 공공데이터가 행정·현장 의사결정에 직접 활용되는 사례를 만든다.

3. **지역 안전 격차 완화**  
   대형 관제 시스템 구축이 어려운 중소 지자체도 웹 기반 SaaS 형태로 위험 분석 기능을 활용할 수 있어 지역 간 산불 대응 역량 격차를 줄일 수 있다.

4. **과잉 경보로 인한 신뢰 저하 방지**  
   false-positive review와 confidence margin을 함께 표시해 무조건적인 경보가 아니라 검토 가능한 위험 판단을 제공한다. 이는 현장 담당자의 서비스 수용성을 높인다.

5. **기후위기 대응형 산림 관리 기반 마련**  
   건조·강풍·고온 등 산불 위험 요인이 증가하는 상황에서, 축적된 산림공공데이터와 실시간 데이터를 결합해 예방 중심의 산림 관리 체계를 강화할 수 있다.

## 7. 기타(관련 실적 및 참고 사항 등)

### 7.1 공모작 개발 히스토리

본 프로젝트는 산림공공·빅데이터를 활용한 산불 대응 서비스 기획에서 출발하여, 실제 웹서비스 구조로 구현을 진행했다. 초기에는 위치 입력 기반 산불 위험 분석을 중심으로 설계했으며, 이후 단순 위험도 산출을 넘어 **오탐 검토**, **confidence margin**, **XAI 설명**, **사용자 유형별 행동 가이드**, **반경 기반 분석**을 포함하는 에이전트형 의사결정 지원 서비스로 확장했다.

주요 개발 흐름은 다음과 같다.

| 단계 | 주요 내용 |
|---|---|
| 데이터 탐색 | 산불 발생 경향성 데이터, 전국 산불 발생 위치도, 오늘 산불 API, AI Hub 샘플 데이터 확인 |
| 데이터 전처리 | CSV/JSON 데이터를 서비스 입력 구조에 맞게 전처리하고 preview/fallback 데이터 구성 |
| 백엔드 구현 | FastAPI 기반 `/api/analyze` 구현, historical/realtime service, risk engine, false-positive review 구성 |
| 에이전트 워크플로우 구현 | LangGraph 기반 select_sources → fetch → 검토 → 보정 → 응답 생성 흐름 구성 |
| 프론트엔드 구현 | Next.js 기반 지도/입력 폼/결과 카드/오탐 패널/대응 가이드 화면 구성 |
| 안정성 보완 | OpenAI API 사용 불가 상황에서도 rule fallback으로 분석이 계속되도록 설계 |

### 7.2 구현 결과 참고 사항

현재 구현된 서비스는 다음 기능을 포함한다.

- 위치 및 반경 기반 산불 분석 요청
- 과거 산불 이력 기반 주변 위험 맥락 조회
- 실시간 산불 API 연계 및 fallback 처리
- 위험도, 위험 점수, confidence, confidence margin 산출
- 오탐 가능성 검토
- XAI 근거 및 검토 신호 제공
- 데이터 품질 요약 제공
- 사용자 유형별 대응 행동 가이드 제공
- 지도 기반 좌표 선택 및 반경 표시 구조
- LLM 사용 불가 시 rule fallback 운영
- **[수정 반영] model score / model confidence / feature coverage를 포함한 baseline model adapter 주입 구조**
- **[수정 반영] model-rule disagreement, 낮은 model confidence, 낮은 feature coverage를 반영한 low-confidence routing**
- **[수정 반영] reroute_applied, reroute_reason, confidence_reasons를 포함한 보수적 secondary-analysis 결과 반환**

### 7.3 향후 고도화 계획

1. **실모델 로더 연결 및 규칙-모델 융합 고도화**  
   현재는 주입 가능한 baseline model adapter와 env stub scorer까지 구현된 상태이며, 향후에는 joblib/LightGBM/XGBoost 기반 실모델 로더를 연결해 지역·계절별 예측 정밀도를 높인다.

2. **위성·드론·CCTV 영상 기반 탐지 결과 연계**  
   영상 기반 연기·화점 탐지 결과를 현재 에이전트 판단 흐름에 추가하여 오탐 검토의 정밀도를 높인다.

3. **기상청 예보·특보 데이터 연동**  
   현재·과거 데이터 중심 분석에서 초단기예보, 건조특보, 강풍특보를 결합해 사전 예측 기능을 강화한다.

4. **지자체 맞춤형 관제 대시보드 확장**  
   행정구역별 취약지, 관할 산림자원, 대피소, 도로망, 소방서 접근성을 결합해 지자체별 의사결정 화면을 제공한다.

5. **오탐 데이터 축적 및 에이전트 성능 개선**  
   실제 현장 검증 결과를 피드백 데이터로 축적하여 false-positive review와 confidence 산정 로직을 지속 개선한다.

6. **보고서 자동 생성 기능**  
   분석 결과를 기반으로 지자체 내부 보고서, 계절별 위험 리포트, 사전 점검 계획서를 자동 생성하는 기능으로 확장한다.

### 7.4 제출 시 유의 사항

- AI Hub 데이터의 공식 목록명은 제출 전 원본 페이지에서 최종 확인하는 것이 바람직하다.
- 예상 매출액은 실제 계약 실적이 아닌 사업화 가정치이므로, 최종 제출 시 팀의 사업 전략에 맞춰 조정 가능하다.
- OpenAI API credit 소진 상황에서는 rule fallback으로 시연 가능하나, LLM source selection 기능을 시연하려면 API 사용 가능 상태를 별도로 확보해야 한다.

1. 본 서비스는 산불 예측·감지·의사결정에 AI 에이전트를 도입해 단순 점수 제공을 넘어 검토·보정·설명·행동 안내까지 자동화한다.
